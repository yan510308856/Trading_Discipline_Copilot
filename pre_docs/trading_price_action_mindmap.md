# 价格行为交易导图

> 目标：把价格分析学习内容整理成交易时可参照的执行框架，用于规范判断流程与交易纪律。后续也要方便用 Codex 转成实用工具，用于交易前检查、交易中提醒、交易后复盘。

---

## 0. 工具化目标：未来用于 Codex 生成交易纪律工具

这份导图不仅是学习笔记，而是未来要被整理成一个实用工具，用来在交易前、交易中、交易后提醒自己保持纪律。

未来工具的核心目标：

1. **交易前检查**：在入场前强制回答关键问题，避免因为情绪、FOMO、主观预判而交易。
2. **交易中提醒**：根据当前市场状态提醒容易忽略的风险，例如无 FT 突破、TR 第二段陷阱、逆势方仍能赚钱、无止损下单、止损后情绪性反手、强突破后留下缺口等。
3. **交易后复盘**：记录这笔交易是否遵守导图规则，区分“策略内亏损”和“纪律性错误”。
4. **规则结构化**：每条知识都尽量整理成可被程序读取的形式，而不是只写成抽象感想。

每次新增知识点都尽量按以下格式整理，方便之后用 Codex 转成工具：

```yaml
rule:
  id: 简短规则编号
  name: 规则名称
  category: 所属模块
  market_context: 适用市场环境
  trigger: 触发条件
  checklist: 交易前/交易中需要检查的问题
  action: 符合条件时的执行建议
  avoid: 需要避免的行为
  risk: 忽略该规则的风险
  discipline_sentence: 一句话纪律
```

后续每条规则都要尽量回答三个问题：

- **什么时候触发？**
- **触发后我应该做什么？**
- **如果忽略它，最可能犯什么错？**

## 1. 使用方式

每次交易前，不只问“现在涨还是跌”，而要按顺序问：

1. 当前市场环境是什么？
2. 价格结构是否清楚？
3. 多空双方分别处在什么位置？
4. 谁在赚钱，谁被套，谁可能被迫止损？
5. 我的入场、止损、出场是否都有结构依据？
6. 这笔交易是否符合纪律，而不是情绪交易？

---

## 2. 市场环境判断

### 2.1 基本状态

- 趋势
- 震荡区间（Trading Range, TR）
- 宽通道趋势
- 窄通道趋势
- 假突破
- 强突破后进入新市场周期

### 2.2 趋势强弱

不能只看价格方向，还要看逆势交易者是否仍然有赚钱空间。

- **强上涨趋势**：逆势空头通常很难赚钱，容易被持续套住并止损。
- **弱上涨趋势 / 宽通道上涨**：虽然价格在上涨，但空头在高位 limit order 做空后仍能通过回调赚钱。
- **强下跌趋势**：逆势多头通常很难赚钱，容易被持续套住并止损。
- **弱下跌趋势 / 宽通道下跌**：虽然价格在下跌，但多头在低位 limit order 做多后仍能通过反弹赚钱。

一句话纪律：

> 强趋势中，逆势一方应该很难赚钱；如果逆势一方也能赚钱，说明趋势不强。

## 3. 价格结构

### 3.1 趋势方向

- 高点和低点是否不断抬高？
- 高点和低点是否不断降低？
- 是否只是区间内来回波动？
- 当前价格是否接近区间上沿或下沿？

### 3.2 突破与 Follow-through

突破不能只看“价格是否穿过关键位”，还要看突破后市场是否接受这个新价格区域。

#### 规则：突破要有好的跟随（Follow-through, FT）

> 好突破不是只有一根突破 K，而是突破之后要有好的跟随。

判断问题：

- 突破 K 线是否足够强？
- 突破后下一根或随后几根 K 线是否继续朝突破方向推进？
- 突破后是否停滞、重叠、快速回到区间内？
- 突破后回调是否很深，甚至直接吞回突破 K？
- 突破后对手方是否仍然能赚钱？

交易含义：

- **有强 FT**：说明市场接受突破方向，趋势延续概率更高，可以考虑顺势跟随或等回踩入场。
- **没有 FT**：说明突破可能只是试探、扫流动性或假突破，不能因为“突破了”就追单。
- **突破后快速失败**：说明被突破的一方没有真正崩溃，反而可能形成反向交易机会。

一句话纪律：

> 突破不是信号，突破后的 FT 才是确认。

### 3.3 突破后回测突破点与缺口

#### 规则：突破后观察是否回测突破点；留下缺口说明趋势可能很强

> 突破之后，如果价格没有回测突破点，甚至在突破点附近留下缺口，说明突破方向很强，市场没有给反方向交易者舒服退出或重新入场的机会。

判断问题：

- 突破后价格是否回测了突破点？
- 回测是很深、很浅，还是根本没有回测？
- 突破点附近是否留下了缺口，价格始终没有回补？
- 突破后是否持续形成顺势 K 线，而不是马上重叠或回到区间？
- 被突破的一方是否被迫追着止损，而不是有机会从容退出？

交易含义：

- **突破后深回测突破点**：说明突破方向仍需要证明自己，不能过早认定为强趋势。
- **突破后浅回测或不回测**：说明对手方压力较大，趋势方向更强。
- **突破后留下缺口且不回补**：说明市场急于接受新价格，突破方向很强，逆势交易要谨慎。
- **缺口后来被快速回补**：强趋势判断需要下降，重新按普通突破或失败突破观察。

工具提醒语：

> 突破后是否回测了突破点？如果没有回测并留下缺口，这可能不是普通突破，而是强趋势信号；不要轻易逆势。

一句话纪律：

> 突破后不回测并留下缺口，是强趋势信号；顺势优先，逆势谨慎。

### 3.4 震荡区间与第二段陷阱

震荡区间中，市场经常给出看似要启动的第二段走势，但第二段未必代表趋势开始，反而可能是陷阱。

#### 规则：震荡区间谨防第二段陷阱（2nd Leg Trap）

> 在 TR 中，第一段突破或冲击之后，第二段跟随如果缺乏力度，可能只是诱多/诱空，而不是趋势开始。

判断问题：

- 当前是否仍在震荡区间内部或接近区间边缘？
- 第一段走势是否只是从区间一侧冲到另一侧？
- 第二段走势是否发生在高位追多或低位追空的位置？
- 第二段是否缺乏强 K、缺乏连续性、成交/动能不足？
- 第二段之后是否马上被反向 K 线打回？

交易含义：

- TR 中不要把每个第二段都当成趋势延续。
- 区间上沿附近的第二段上涨，可能是诱多。
- 区间下沿附近的第二段下跌，可能是诱空。
- 如果第二段没有好的 FT，要警惕失败突破和反向回归区间。

一句话纪律：

> 震荡区间中，第二段常常不是机会，而是陷阱。

### 3.5 震荡区间中的大阳线 / 大阴线

在震荡区间中，大阳线和大阴线不一定代表趋势启动。相反，越是在区间上沿或下沿出现的强 K，越要警惕其成为反转信号。

#### 规则：震荡区间中，大阳线大阴线注意反转

> TR 中强 K 不一定是趋势确认，可能是最后一冲、诱多/诱空，或测试区间边缘后的反转起点。

判断问题：

- 当前是否处在震荡区间，而不是强趋势？
- 大阳线是否出现在区间上沿、前高附近或阻力位附近？
- 大阴线是否出现在区间下沿、前低附近或支撑位附近？
- 强 K 之后是否缺乏 follow-through？
- 强 K 后下一根是否出现反向 K、长影线、重叠或快速吞回？
- 这根强 K 是否让追单的一方进入不舒服的位置？

交易含义：

- **TR 上沿的大阳线**：不要自动理解为强多头启动；如果没有 FT，可能是诱多，随后反转回区间。
- **TR 下沿的大阴线**：不要自动理解为强空头启动；如果没有 FT，可能是诱空，随后反转回区间。
- 在 TR 中，强 K 后更重要的是看市场是否继续接受该方向，而不是只看单根 K 的力度。
- 如果强 K 出现在区间极端位置但没有跟随，要优先想到失败突破、反向交易或至少避免追单。

一句话纪律：

> TR 中强 K 不是追单理由；区间极端位置的强 K，没有 FT 就优先警惕反转。

### 3.6 Inside Bar 与收敛三角形：大概率突破模式

Inside bar、连续 inside bar、以及逐步收敛的三角形结构，通常代表价格波动范围被压缩，多空双方暂时都无法有效推进。压缩越明显，后续出现突破的概率越高。

#### 规则：inside bar 收敛三角形，大概率突破模式

> Inside bar / 收敛三角形不是直接判断方向的信号，而是提醒市场正在压缩，之后大概率会选择方向突破。

判断问题：

- 当前 K 线是否被前一根 K 线完全包含，形成 inside bar？
- 是否连续出现 inside bar，说明波动范围持续变窄？
- 高点是否逐渐降低，低点是否逐渐抬高，形成收敛三角形？
- 收敛结构发生在趋势中、区间中部，还是关键支撑阻力附近？
- 突破发生后是否有好的 follow-through？
- 突破方向是否让对手方被套、止损或被迫追单？

交易含义：

- Inside bar / 收敛三角形本质是压缩，不是方向确认。
- 压缩后容易突破，但突破方向需要等待市场选择。
- 不要在收敛结构内部频繁交易，因为空间越来越小，容易被来回扫。
- 真正的机会通常在突破后：看突破 K 的质量和后续 FT。
- 如果突破后没有 FT，要警惕假突破，尤其是在震荡区间或区间边缘。
- 如果突破后有强 FT，可以把它作为趋势启动、趋势延续或区间突破的候选信号。

一句话纪律：

> Inside bar / 三角形是压缩信号，不是方向信号；等突破，等 FT，再交易。

### 3.7 昨日高点 / 昨日低点的两次失败突破：可能反转

昨日最高点和昨日最低点是非常重要的参考价位，因为很多交易者会围绕它们设置突破单、止损单、止盈单和反向 limit order。

如果价格两次尝试突破昨日高点或昨日低点都失败，说明市场暂时不接受突破后的新价格区域。此时要警惕失败突破后的反转。

#### 规则：两次尝试突破昨天最高或最低点都失败，有可能反转

> 昨日高低点附近的两次突破失败，说明突破方向没有足够力量，追突破的一方可能被套，市场可能反向回到区间或走出反转。

判断问题：

- 当前价格是否正在测试昨日最高点或昨日最低点？
- 第一次突破是否很快失败，价格回到昨日高/低点内侧？
- 第二次尝试是否仍然缺乏好的 follow-through？
- 第二次突破后是否出现反向 K、长影线、重叠或快速吞回？
- 突破方向的交易者是否开始被套？
- 反向交易者是否开始赚钱，并可能推动回归区间？

交易含义：

- **两次上破昨日高点失败**：不要继续无脑追多；要警惕多头突破失败后止损，形成向下反转。
- **两次下破昨日低点失败**：不要继续无脑追空；要警惕空头突破失败后止损，形成向上反转。
- 第一次失败可能只是测试，第二次失败更能说明突破方向力量不足。
- 如果第二次失败后出现强反向 K 或好的反向 FT，可以把它视为反转候选信号。
- 如果仍处于震荡环境，则两次失败突破更容易变成回归区间的交易机会。

工具提醒语：

> 价格已经两次尝试突破昨日高/低点失败。你是在追一个已经失败两次的突破，还是应该开始警惕反转？

一句话纪律：

> 昨日高低点两次突破失败，不再追突破方向；先警惕反转，再等待确认。

## 4. 多空双方处境分析

这是当前框架中的核心模块。

### 4.1 多头处境

交易前要问：

- 多头现在是否在赚钱？
- 多头是否被套？
- 多头止损大概在哪里？
- 多头是否有动力继续追买？
- 多头是否已经接近目标位，可能开始止盈？

### 4.2 空头处境

交易前要问：

- 空头现在是否在赚钱？
- 空头是否被套？
- 空头止损大概在哪里？
- 空头是否有动力继续加空？
- 空头是否已经接近目标位，可能开始止盈？

### 4.3 双方处境的判断规则

#### 规则：趋势方向不等于趋势强度

> 虽然是上涨趋势，但是 limit order 空头在赚钱，说明空头不弱，多头也没有强到完全压制空头。

含义：

- 表面上价格在上涨；
- 但空头可以在高位挂 limit order 做空；
- 并且通过随后的回调获利；
- 说明市场不是强单边多头；
- 更可能是弱上涨、宽通道上涨，或者接近震荡区间。

交易含义：

- 不能因为“看起来在涨”就无脑追多；
- 如果空头也能赚钱，说明市场有双向交易空间；
- 此时更适合等回调、等确认，或按区间/宽通道逻辑交易；
- 入场前必须判断多空双方谁更舒服、谁更痛苦。

一句话纪律：

> 不要只看价格往哪里走，要看多空双方分别处在什么处境。

## 5. 入场逻辑

### 5.1 入场前必须回答

- 我为什么现在入场？
- 这笔交易是趋势跟随、回调入场，还是反转交易？
- 我的依据是价格结构，还是情绪冲动？
- 如果我错了，市场会在哪里证明我错？

### 5.2 强趋势中的入场

强趋势中，逆势一方通常很难赚钱。

- 强上涨趋势：优先考虑顺势多头。
- 强下跌趋势：优先考虑顺势空头。
- 不轻易逆势摸顶/抄底。

### 5.3 弱趋势 / 宽通道中的入场

如果逆势 limit order 交易者也能赚钱，则趋势并不强。

- 不适合无脑追突破。
- 更适合等待回调。
- 更重视区间边缘、前高前低、支撑阻力。
- 止盈需要更保守。

### 5.4 突破交易入场过滤

突破入场前必须先确认：

1. 突破发生在什么市场环境中：趋势、宽通道，还是震荡区间？
2. 突破后是否有好的 FT？
3. 突破后是否回测突破点，是否留下缺口？
4. 突破方向的一方是否在赚钱并继续加仓？
5. 被突破的一方是否被套、止损、被迫退出？
6. 如果没有 FT，我是否正在追一个可能失败的突破？

执行规则：

- **突破 + 好 FT**：可以考虑顺势跟随，或等待回踩后继续顺势。
- **突破 + 无 FT**：不追，等待确认；如果快速回到区间内，按失败突破处理。
- **突破后不回测突破点 / 留下缺口**：说明趋势可能很强；顺势交易优先，逆势交易要极其谨慎。
- **突破后留下缺口但随后快速回补**：降低强趋势判断，重新观察是否变成普通回调或失败突破。
- **TR 中第二段 + 无强 FT**：谨防 2nd Leg Trap，不在区间极端位置追单。

### 5.5 震荡区间强 K 入场过滤

在 TR 中看到大阳线或大阴线时，入场前必须先判断它发生的位置。

执行规则：

- **区间中部的强 K**：通常信息不够明确，容易进入来回震荡，谨慎追单。
- **区间上沿的大阳线**：除非后续有强 FT，否则不追多；优先警惕诱多或失败突破。
- **区间下沿的大阴线**：除非后续有强 FT，否则不追空；优先警惕诱空或失败突破。
- **强 K 后没有 FT**：不把它当作趋势信号，等待确认或反向结构。

工具提醒语：

> 你现在是否在 TR 上沿追大阳线，或在 TR 下沿追大阴线？如果没有好的 FT，这可能不是趋势启动，而是反转陷阱。

### 5.6 Inside Bar / 收敛三角形突破入场过滤

看到 inside bar 或收敛三角形时，不能提前主观预测方向，而要先把它识别为“压缩状态”。

执行规则：

- **压缩内部**：不频繁交易，避免在小空间里被反复扫损。
- **向上突破 + 好 FT**：可以考虑顺势多头，或等待突破回踩确认。
- **向下突破 + 好 FT**：可以考虑顺势空头，或等待突破回踩确认。
- **突破 + 无 FT**：不追，优先按假突破风险处理。
- **TR 中的收敛突破**：必须额外警惕失败突破，因为震荡区间天然容易回归区间。

工具提醒语：

> 你看到的是方向信号，还是压缩信号？Inside bar / 三角形只说明要突破，不说明一定往哪边突破；等突破和 FT。

### 5.7 昨日高低点两次失败突破入场过滤

昨日高点和昨日低点属于高关注度位置。看到价格突破这些位置时，不能只问“是否突破”，而要问“突破是否被接受”。

执行规则：

- **第一次突破昨日高/低点失败**：先降低追突破的信心，等待第二次测试或更明确的 FT。
- **第二次突破昨日高/低点仍失败**：优先按失败突破风险处理，不继续追突破方向。
- **两次上破昨日高点失败 + 强反向空头 K**：警惕向下反转，可以观察做空候选，但仍要看反向 FT。
- **两次下破昨日低点失败 + 强反向多头 K**：警惕向上反转，可以观察做多候选，但仍要看反向 FT。
- **两次失败但没有强反向确认**：不急着反手，至少避免继续追原突破方向。

工具提醒语：

> 昨日高/低点已经两次突破失败。继续追突破方向的胜率是否已经下降？是否应该等待反向确认或放弃交易？

## 6. 风险控制

### 6.1 止损逻辑

止损不应该随意设置，而应放在结构失效的位置。

- 多头止损：通常放在关键低点下方。
- 空头止损：通常放在关键高点上方。
- 如果价格到达该位置，说明原交易逻辑失效。

### 6.2 仓位纪律

- 趋势强弱不清楚时，仓位应降低。
- 如果市场允许多空双方都赚钱，不应使用强趋势交易仓位。
- 亏损后不能因为情绪加仓。

### 6.3 下单前必须设置止损

止损不是亏损后才临时决定的动作，而是下单前就必须确定的交易条件。

#### 规则：下单一定要设置好止损

> 没有止损，就没有入场资格。每一笔订单都必须先知道：如果错了，在哪里认错。

判断问题：

- 我是否已经设置好止损单，而不是只在脑中想一个止损？
- 止损位置是否对应结构失效点？
- 止损距离是否过大，导致这笔交易风险不可接受？
- 如果止损被打掉，我是否能够接受这笔亏损，而不是临时移动止损？
- 这笔交易的仓位是否基于止损距离计算，而不是凭感觉下单？

交易含义：

- 入场前必须先确定 invalidation point。
- 止损应该服务于交易逻辑，而不是服务于“我不想亏”。
- 如果找不到合理止损，这笔交易就不应该做。
- 如果止损距离太大，要么降低仓位，要么放弃交易。

工具提醒语：

> 你还没有设置结构性止损。没有止损，不允许下单。

### 6.4 止损后的冷静期

止损之后最容易出现情绪化交易：不服气、想立刻赚回来、看到价格反向就马上反手。这类交易通常不是基于结构，而是基于亏损后的情绪反应。

#### 规则：止损之后不要立刻反向开仓，先冷静下来认真思考

> 被止损只说明上一笔交易逻辑失效，不自动说明反方向交易成立。

判断问题：

- 我现在想反向开仓，是因为出现了新的结构信号，还是因为刚刚被止损？
- 反向开仓是否有独立入场理由？
- 市场是否真的完成方向转换，还是只是正常回调 / 扫损？
- 我是否因为不服气、着急回本、害怕错过而想立刻交易？
- 如果这不是刚止损后的第一分钟，我还会做这笔反向单吗？

交易含义：

- 止损后先暂停，不立刻反手。
- 重新评估市场环境、价格结构、多空双方处境。
- 只有当反方向出现独立结构依据时，才把它当作一笔新的交易。
- 反向开仓必须重新完成完整 checklist：入场理由、止损、目标、风险。

工具提醒语：

> 你刚刚被止损。现在禁止情绪性反手；先重新判断市场结构，再决定是否有新的交易。

## 7. 出场逻辑

### 7.1 止盈判断

- 是否到达 MM 目标位？
- 是否接近前高 / 前低？
- 是否接近区间上沿 / 下沿？
- 是否出现对手方重新变强的信号？

### 7.2 弱趋势中的止盈纪律

如果市场是弱趋势或宽通道：

- 不要贪心；
- 到关键位附近要考虑保护利润；
- 不要为了最后一点利润放弃已经获得的优势。

## 8. 交易纪律总表

### 8.1 当前已加入的纪律

- 不要只判断趋势方向，要判断趋势强度。
- 不要只看多头是否赚钱，也要看空头是否同样能赚钱。
- 强趋势中，逆势一方应该很难赚钱。
- 如果逆势一方也能赚钱，说明趋势不强。
- 趋势不强时，按弱趋势、宽通道或震荡逻辑处理。
- 交易前必须判断双方处境：谁舒服，谁痛苦，谁可能被迫止损。
- 突破必须观察 follow-through，不要只因价格穿过关键位就追单。
- 突破后要观察是否回测突破点；若没有回测并留下缺口，说明趋势可能很强，逆势交易要谨慎。
- 震荡区间中，第二段走势经常是陷阱；没有强 FT 时，不在区间上沿追多或区间下沿追空。
- 震荡区间中，大阳线和大阴线要结合位置判断；区间上沿的大阳线、区间下沿的大阴线如果没有 FT，优先警惕反转。
- Inside bar / 收敛三角形是压缩信号，不是方向信号；等突破，等 FT，再交易。
- 昨日高点 / 昨日低点附近，两次突破失败要警惕反转；不要在已经失败两次的突破方向继续冲动追单。
- 下单前必须设置好止损；没有止损，就没有入场资格。
- 止损必须对应结构失效点，而不是亏损后临时决定。
- 止损之后不要立刻反向开仓；先冷静下来，重新判断市场结构。
- 反向开仓必须有独立的新交易逻辑，不能只是因为刚刚被止损。


- 到达前高/前低、整数位、EMA、50% 回撤、区间边缘、MM 或缺口时，必须重新评估，不无脑追单。
- 突破后要用 MM / Leg 等幅 / 实际风险规划目标，不靠幻想止盈。
- 强趋势可能无视支撑阻力；窄通道中的第一次反转大概率失败。
- 开盘前 6-12 根 K 常形成 Opening Range，开盘趋势至少需要连续强 K 和 FT 证明。
- 大幅跳空后重点观察 EMA、缺口 50%、双顶双底和楔形结构，不只看跳空方向。
- 夜盘高低点是日内重要流动性锚点，突破与失败突破都要重点观察。
- 三推楔形、抛物线楔形和 final flag 常提示衰竭或趋势中继，第三推后不盲目追。
- 缺口要分类：breakout gap、measuring gap、exhaustion gap 的交易含义不同。
- 高质量突破应具备大实体、收在极端、远离区间、急迫感和 FT；弱突破先按假突破风险处理。
- Surprise bar 经常带来第二段行情，不要看到一根大 K 就立刻逆势。
- 信号 K 的位置和市场环境比形态本身更重要。
- BO 有 FT 后，优先等待 pullback，而不是急着猜 reversal。
- 大周期定环境，小周期找入场；不要用 5m 信号硬扛 1h/4h 结构。
- TR/Channel 中突破大概率失败；做失败突破要给结构止损空间，并用仓位控制风险。

### 8.2 一句话纪律

> 不要只看价格往哪里走，要看多空双方分别处在什么处境。  
> 突破看 FT，也看是否回测；不回测并留下缺口时，优先尊重强趋势。  
> 没有止损，不下单；刚被止损，不反手。

## 9. 工具化设计原则

### 9.1 每条交易知识都要转成“可执行规则”

以后不只记录“我学到了什么”，而要进一步整理成：

- 这条规则适用于什么市场环境？
- 什么情况会触发提醒？
- 它要求我暂停、等待、入场、减仓、止盈，还是禁止交易？
- 它主要防止哪一种纪律错误？

### 9.2 工具中的核心提醒类型

未来交易纪律工具至少应包含以下提醒类型：

1. **方向误判提醒**：不要只看涨跌方向，要判断趋势强弱。
2. **突破过滤提醒**：突破后没有好的 FT，不把突破当作有效趋势。
3. **突破强度提醒**：突破后是否回测突破点；不回测并留下缺口时，提醒趋势可能很强。
4. **震荡陷阱提醒**：TR 中第二段、区间极端位置的大阳线/大阴线都容易诱多/诱空，避免无 FT 追单。
5. **压缩突破提醒**：inside bar / 收敛三角形是等待突破的压缩状态，不提前主观押方向。
6. **关键位失败突破提醒**：昨日高点 / 昨日低点两次突破失败时，提醒不要继续追突破方向，并警惕反转。
7. **多空处境提醒**：判断谁在赚钱、谁被套、谁可能被迫止损。
8. **入场纪律提醒**：入场必须有结构依据，不能只是觉得“要涨/要跌”。
9. **止损纪律提醒**：止损必须对应结构失效点，而不是随意设置。
10. **止损后冷静提醒**：止损后禁止情绪性反手，必须重新完成 checklist。
11. **出场纪律提醒**：弱趋势或区间中不要贪心，到关键位要保护利润。
12. **MM / 目标位提醒**：突破后是否已计算 MM、Leg 等幅、1R/2R 与实际风险。
13. **开盘模式提醒**：开盘前 6-12 根 K 是否仍是 Opening Range；第一根 5m 高低点是否被有效突破。
14. **跳空开盘提醒**：Big Gap Up/Down 后是否接近 EMA、缺口 50%、双顶双底或楔形。
15. **夜盘高低点提醒**：夜盘高低点是否被突破，是否出现失败突破。
16. **楔形/Final Flag 提醒**：是否出现三推、抛物线楔形或趋势末端小旗形。
17. **缺口分类提醒**：当前 gap 是突破型、测量型还是衰竭型，是否被回补。
18. **信号 K 位置提醒**：信号 K 是否出现在关键位置，而不是区间中部。
19. **Pullback 提醒**：BO 有 FT 后优先等待 PB，不急着猜反转。
20. **多周期提醒**：1h/4h 定环境，5m 找入场，避免小周期硬扛大周期。
21. **复盘提醒**：每笔交易结束后记录是否违反规则。

### 9.3 未来工具的交易前检查模板

```text
交易前检查：

1. 当前市场环境是什么？
   - 强趋势 / 弱趋势 / 宽通道 / 震荡区间 / 假突破？

2. 价格是否在关键位置？
   - 前高、前低、区间上沿、区间下沿、突破位、回踩位、昨日高低点？

3. 突破是否有好的 follow-through？
   - 如果没有 FT，我是否正在追一个可能失败的突破？

4. 突破后是否回测突破点？
   - 是否不回测并留下缺口？
   - 如果留下缺口，我是否正在逆势交易一个很强的突破？

5. 昨日高/低点是否已经两次突破失败？
   - 如果两次上破昨日高点失败，我是否还在追多？
   - 如果两次下破昨日低点失败，我是否还在追空？
   - 第二次失败后是否出现反向 K 或反向 FT？

6. 是否处在震荡区间第二段陷阱中？
   - 我是否在区间上沿追多，或在区间下沿追空？

7. 是否在震荡区间极端位置追强 K？
   - 我是否在区间上沿追大阳线，或在区间下沿追大阴线？
   - 强 K 后是否真的有好的 follow-through？

8. 是否看到 inside bar / 收敛三角形？
   - 这是压缩信号还是方向信号？
   - 我是否在突破前主观猜方向？
   - 突破后是否有好的 follow-through？

9. 多空双方处境如何？
   - 谁在赚钱？谁被套？谁会被迫止损？

10. 我的入场理由是什么？
   - 是结构信号，还是情绪冲动？

11. 我的止损在哪里？
   - 是否已经设置好止损单？
   - 止损是否对应结构失效点？
   - 到达哪里说明交易逻辑失效？

12. 我的目标位在哪里？
   - 到关键位是否需要保护利润？

13. 我是否刚刚被止损？
   - 如果是，我现在想交易是新的结构机会，还是情绪性反手？
   - 是否需要先暂停、冷静、重新判断？

14. 这笔交易违反了哪条纪律吗？
   - 如果违反，是否应该直接放弃？
```

```yaml
      - 大阴线是否出现在区间下沿或支撑位附近？

      - 强 K 是否可能只是诱多/诱空，让追单方被套？
    action: 不因单根强 K 直接追单；等待 FT、回踩确认，或观察失败后的反向机会
    avoid: 在 TR 上沿追大阳线，或在 TR 下沿追大阴线
    risk: 把区间极端位置的最后一冲误判为趋势启动，追在反转前
    discipline_sentence: TR 中强 K 不是追单理由；区间极端位置的强 K，没有 FT 就优先警惕反转。

  - id: inside_bar_triangle_breakout_setup
    name: Inside bar / 收敛三角形是大概率突破模式
    category: 压缩突破结构
    market_context: Inside bar, 连续 inside bar, 收敛三角形, 趋势中继, 震荡区间中部或边缘

      - 真正突破后是否有好的 FT？

    avoid: 在压缩内部频繁交易，或在突破前主观押方向

  - id: yesterday_high_low_two_failed_breakouts

    category: 关键位失败突破

      - 第二次突破是否仍然缺乏好的 FT？

      - 反向交易者是否开始赚钱？

    avoid: 在昨日高/低点已经两次失败突破后继续追多或追空
    risk: 追入关键位失败突破，被突破方止损和反向交易共同推动的反转打损
    discipline_sentence: 昨日高低点两次突破失败，不再追突破方向；先警惕反转，再等待确认。

    avoid: 被止损后不服气、报复交易、立即反手、加倍下单
```

## 12. 从《价格行为学笔记.pdf》整合的新增规则

> 本节来自旧学习笔记的二次提炼。处理原则：不逐字抄笔记，而是把有交易价值的内容转成可执行规则、检查问题和工具提醒。

### 12.1 常见支撑、阻力与目标位

价格到达关键位时，不应只看单根 K 线，而要先判断它是否靠近常见支撑/阻力或目标位。

常见关键位：

- 前期高点 / 前期低点；
- 重要整数位；
- EMA / 均线；
- 宽通道或震荡区间的上沿、下沿；
- 前一段行情的 50% 回撤位；
- 多空双方的打平离场点；
- Measured Move（MM）目标位；
- 缺口区域，尤其是 body gap、breakout gap、measuring gap。

交易含义：

- 突破后优先寻找 MM 目标位，不要只凭感觉止盈。
- 到达 MM、前高前低、整数位、EMA、通道边缘时，要重新评估是否保护利润。
- 强趋势可能暂时无视支撑/阻力；弱趋势或 TR 中，支撑/阻力更容易生效。

工具提醒语：

> 当前价格是否接近前高/前低、整数位、EMA、50% 回撤、区间边缘或 MM？如果是，不能无脑追单，要先判断趋势强度与双方处境。

一句话纪律：

> 关键位不是自动反转点，但必须触发重新评估。

### 12.2 Measured Move 与 Leg 等价止盈

MM 是旧笔记中反复出现的止盈/目标位方法。它适合用于突破后、N 字结构、强趋势第二段行情等场景。

常见用法：

- **突破后找 MM**：突破区间、开盘区间、震荡区间后，估算等幅目标。
- **Leg 1 = Leg 2**：第二段走势常常接近第一段走势的幅度，即 AB = CD。
- **实际风险 Actual Risk**：用入场点到结构止损点的真实风险，规划 1R、2R 等目标。

交易含义：

- 强突破后，如果有好的 FT，大概率至少会有第二段行情，可以用 MM 作为目标候选。
- 到达 MM 或 1R/2R 附近时，不要继续幻想无限延伸，先考虑保护利润。
- 如果实际风险太大，即使形态好，也要降低仓位或放弃。

工具提醒语：

> 这笔交易的 MM 目标在哪里？实际风险是多少？到 1R/2R 或 Leg 等幅位置时，是否应该保护利润？

一句话纪律：

> 入场前先知道目标和实际风险；突破后用 MM，不靠幻想止盈。

### 12.3 强趋势中不要轻易逆势抄底/摸顶

旧笔记强调：强下跌趋势往往会无视支撑位；强上涨趋势也会无视阻力位。不能因为价格到了“看起来便宜/贵”的位置就逆势交易。

判断问题：

- 是否处在窄通道趋势或连续强 K 趋势中？
- 逆势一方是否持续失败，并且很难赚钱？
- 第一次反转信号是否太早、太弱、缺乏 FT？
- 价格是否虽然到达支撑/阻力，但仍然没有出现真正反向力量？

交易含义：

- 强下跌趋势中，支撑位可能被直接打穿；不要因为“到支撑了”就抄底。
- 强上涨趋势中，阻力位可能被直接突破；不要因为“到阻力了”就摸顶。
- 窄通道中的第一次反转大概率失败，更适合等待第二次入场、微型双底/双顶、楔形或更明确的反向 FT。

工具提醒语：

> 当前是否是强趋势/窄通道？如果是，第一次逆势反转信号很可能失败；不要因为到支撑/阻力就直接逆势。

一句话纪律：

> 强趋势先尊重趋势，支撑阻力只是提醒，不是逆势开仓理由。

### 12.4 早盘交易：Opening Range、Opening Reversal、Breakout Mode

旧笔记中对开盘阶段有较多内容，适合转成交易工具中的“开盘模式识别”。

#### Opening Range

- 大多数交易日，开盘前 6-12 根 K 线容易形成震荡区间。
- 尤其是大幅跳空开盘后，价格经常回调或反弹到 EMA 附近。
- 如果要判断开盘趋势，至少需要连续两根强趋势 K 线。

#### Breakout Mode

- 可以用第一根 5 分钟 K 线的最高点和最低点作为初始边界。
- 从第二根 5 分钟 K 线开始观察：
  - 尝试突破第一根高点失败，可能向下反转；
  - 尝试跌破第一根低点失败，可能向上反转。
- 突破开盘震荡区间后，可以寻找 MM 目标。

#### Opening Reversal

- 开盘初期如果冲向某一方向后迅速失败，可能形成开盘反转。
- 开盘反转必须结合位置：EMA、昨日高低点、夜盘高低点、前高前低、MM、缺口区域。
- 不要只因为第一段很强就追，开盘阶段容易出现假方向。

工具提醒语：

> 现在是否还处在开盘前 6-12 根 K 的 Opening Range？第一根 5 分钟 K 的高低点是否被有效突破？突破后是否有 FT？

一句话纪律：

> 开盘先识别区间，不急着定方向；开盘趋势至少要有连续强 K 证明。

### 12.5 大幅跳空开盘的四种常见结果

旧笔记整理了 big gap up / big gap down 的常见路径。跳空本身不等于趋势，重点是观察是否回测 EMA、是否形成双顶双底或楔形反转。

#### 大幅高开 Big Gap Up

常见结果：

- 回踩 EMA 附近形成双底后继续上涨；
- 回踩 EMA 附近形成楔形底后继续上涨；
- 形成双顶后向下反转；
- 形成楔形顶后向下反转。

#### 大幅低开 Big Gap Down

常见结果：

- 反弹 EMA 附近形成双顶后继续下跌；
- 反弹 EMA 附近形成楔形顶后继续下跌；
- 形成双底后向上反转；
- 形成楔形底后向上反转。

交易含义：

- 大跳空开盘后，回测 EMA / 缺口 50% 是常见路径。
- 高开后不要只想着追多；低开后不要只想着追空。
- 大跳空后的顶部/底部结构，比单根开盘 K 更重要。

工具提醒语：

> 今天是否大幅跳空？如果是，你现在是在追跳空方向，还是已经接近 EMA/缺口 50%/双顶双底/楔形位置？

一句话纪律：

> 大跳空先等结构：回测 EMA、双顶双底、楔形，比开盘方向更重要。

### 12.6 夜盘高低点与早盘极值概率

旧笔记中提到两个日内统计倾向：

- 夜盘最高点或最低点，大概率至少会有一个在日内被突破。
- 开盘后越往后，日内最高点或最低点已经出现的概率越高：
  - 第 1 根 5 分钟 K 后，约 20% 概率已经出现当天高/低之一；
  - 第 6 根后，约 50%；
  - 第 12 根后，约 70%；
  - 第 18 根后，约 90%。

交易含义：

- 夜盘高低点是重要流动性位置，日内要重点观察突破与失败突破。
- 早盘如果已经出现大幅单向运动，即使之后反转，也不一定会大幅超过早盘极值。
- 越接近日内后半段，越不要随意假设还能轻松创新高/新低。

工具提醒语：

> 夜盘高低点是否已经被突破？当前是第几根 5 分钟 K？日内高低点是否可能已经出现？

一句话纪律：

> 夜盘高低点是流动性锚点；早盘极值越晚越重要。

### 12.7 Wedge、Parabolic Wedge 与 Final Flag

旧笔记中多次出现楔形结构。楔形不是单纯的三角形，而是多次推动后的动能衰竭或趋势中继结构。

常见类型：

- Wedge Top / Wedge Bottom：三推后可能反转。
- Wedge Bull Flag / Wedge Bear Flag：趋势中的楔形回调，可能延续原趋势。
- Truncated Wedge：缩减版楔形，推动不足但仍可提示衰竭。
- Failed BO of Double Top / Double Bottom：突破双顶/双底失败后反向。
- Parabolic Wedge Top / Bottom：斜率越来越大，高点或低点相连，常代表加速后的衰竭。
- Final Flag：趋势末端的最后小旗形，常见 inside-inside 或小型整理，最后一冲后可能反转。

判断问题：

- 是否有三次推动？
- 三推之间是否有重叠？
- 趋势是否正在加速，斜率越来越大？
- 第三推后是否缺乏 FT 或出现反向强 K？
- 这是趋势中的楔形旗形，还是趋势末端的衰竭楔形？

交易含义：

- 有重叠的楔形更容易代表交易双方都参与，反转信号更可信。
- Parabolic wedge 代表加速行情，但加速之后容易衰竭，不能在第三推末端追单。
- 楔形旗形若出现在强趋势回调中，可能不是反转，而是顺势延续信号。

工具提醒语：

> 当前是否已经出现三推楔形？它是趋势回调中的楔形旗形，还是趋势末端的抛物线楔形？第三推后是否还有 FT？

一句话纪律：

> 三推之后不盲目追；先判断是楔形旗形延续，还是楔形衰竭反转。

### 12.8 缺口体系：Body Gap、Breakout Gap、Measuring Gap、Exhaustion Gap

旧笔记将缺口作为趋势强弱、目标位和支撑阻力的重要依据。

缺口定义：

- 图表中任何没有重叠的部分都可以视为缺口，包括最高/最低之间、收盘价之间、EMA/趋势线之间。
- 收盘价之间的缺口最重要，称为 Body Gap。
- EMA 缺口 / Moving Average Gap（MAG）说明价格远离均线，可能代表强趋势，也可能代表后续回归均线压力。

缺口类型：

- Breakout Gap：突破型缺口，说明市场快速脱离原区间。
- Measuring Gap：测量型缺口，可辅助估计目标位。
- Exhaustion Gap：衰竭型缺口，常出现在趋势末端，后续容易反转或深回调。

交易含义：

- 缺口是目标位，也是潜在阻力/支撑。
- 强趋势会不断制造缺口；只要缺口不被回补，趋势仍需被尊重。
- 如果缺口被快速回补，趋势强度下降。
- 趋势末端的缺口不能简单当作继续追单理由，要警惕 exhaustion gap。

工具提醒语：

> 当前缺口是哪一种：breakout gap、measuring gap，还是 exhaustion gap？缺口是否被回补？它现在是目标位还是支撑/阻力？

一句话纪律：

> 缺口不只是空白区域；它是趋势强弱、目标位和支撑阻力的证据。

### 12.9 成功率高的突破特征

旧笔记列出了成功突破的特征，这可以作为突破交易工具的核心过滤器。

高质量突破通常具备：

- 突破 K 体积大；
- 收在最高/最低附近，几乎没有反向影线；
- 离之前区间足够远；
- 被突破的 K 线数量很多；
- 突破 K 形成有急迫感；
- 突破后有好的 FT；
- 突破制造缺口或不回测突破点；
- 对手方被套、止损或被迫追随。

弱突破通常表现为：

- 突破 K 小；
- 影线多；
- 突破后马上重叠；
- 回调很深；
- 没有第二段；
- 很快回到原区间。

工具提醒语：

> 这根突破 K 是大实体、收在极端、远离区间、有急迫感，还是只是小实体带影线的试探？

一句话纪律：

> 只交易高质量突破；小实体、多影线、无 FT 的突破先当作假突破风险。

### 12.10 Surprise Bar 与第二段行情

旧笔记提到：当小概率事件发生时，会形成 pain trade，并经常制造第二段行情。Surprise bar 常常意味着市场周期可能变化。

判断问题：

- 是否出现一根明显超出预期的大 K？
- 这根 K 是否打破原有区间、趋势线或双方预期？
- 追错方向的人是否被迫止损？
- 后续是否出现第二段跟随？

交易含义：

- Surprise bar 不要简单逆势，因为它可能带来第二段行情。
- 如果 surprise bar 打破原来的 TR，说明市场周期可能从 TR 切换到趋势。
- 但在 TR 中，第二段突破也可能是陷阱，所以仍要看 FT、位置和是否回到区间。

工具提醒语：

> 是否出现 surprise bar？这可能意味着市场周期变化，并带来第二段行情；不要马上逆势，也不要忽略 TR 陷阱。

一句话纪律：

> 小概率大 K 出现后，先假设可能有第二段，不要急着逆势。

### 12.11 信号 K 与入场：位置比形态重要

旧笔记强调：信号 K 重要，但所处环境比信号 K 本身更重要。

判断问题：

- 信号 K 出现在什么位置：区间边缘、EMA、昨日高低点、MM、缺口、趋势线，还是区间中部？
- 当前市场是趋势还是 TR？
- 信号 K 是否被长影线干扰？
- 信号 K 后是否有确认 K 或 FT？
- 如果信号 K 形态一般，但位置很好，是否仍然有交易价值？
- 如果信号 K 形态很好，但位置很差，是否应该放弃？

交易含义：

- 不要迷信单根信号 K。
- 位置、市场环境、趋势强弱和 FT 比信号 K 形态更重要。
- 长影线不一定否定信号，但需要看下一根是否确认。

工具提醒语：

> 你交易的是信号 K 的形态，还是它在关键位置上的意义？位置不对，信号 K 再漂亮也要打折。

一句话纪律：

> 先看位置和环境，再看信号 K。

### 12.12 Pullback、H1/H2、L1/L2 与顺势入场

旧笔记中 H1/H2、L1/L2 属于回调入场框架，适合未来工具做顺势入场提醒。

基本含义：

- Pullback：大趋势中的小级别逆向回调。
- H1 / H2：上涨趋势回调中，第一次/第二次尝试恢复上涨的做多信号。
- L1 / L2：下跌趋势反弹中，第一次/第二次尝试恢复下跌的做空信号。

交易含义：

- 强上涨趋势中，小回调后的 H1/H2 可以作为顺势做多候选。
- 强下跌趋势中，小反弹后的 L1/L2 可以作为顺势做空候选。
- 坏的 H1/L1 信号 K 要谨慎：例如 H1 是阴线、L1 是阳线时，顺势信号质量下降，甚至可能成为反向机会。
- 突破后有 FT，再出现回调时，优先考虑 pullback 入场，而不是过早猜反转。

工具提醒语：

> 这是强趋势中的 pullback，还是趋势反转？如果 BO 后有 FT，优先等 PB，而不是先猜 reversal。

一句话纪律：

> BO 有 FT 后，优先交易 pullback，不急着猜反转。

### 12.13 2 Bar Reversal 与微型双顶/双底

旧笔记提到 2 bar reversal：两根 K 线实体相近、方向相反，常作为小级别反转信号。

判断问题：

- 两根 K 是否大小相近、方向相反？
- 是否发生在关键位：EMA、区间边缘、昨日高低点、MM、缺口？
- 第二根 K 是否收得足够强？
- 后续是否有 FT？
- 它是强趋势中的普通回调，还是关键位上的反转？

交易含义：

- 2 bar reversal 只有在关键位置更有价值。
- 强趋势中，2 bar reversal 可能只是小回调，不应过度解读。
- 与微型双顶/双底、H1/H2、L1/L2结合时，信号更清楚。

工具提醒语：

> 这是关键位上的 2 bar reversal，还是强趋势中的普通小回调？没有位置和 FT，不要过度解读。

一句话纪律：

> 2 bar reversal 必须结合位置；形态本身不够。

### 12.14 多周期判断：大周期定环境，小周期找入场

旧笔记中提到用 1h、4h 判断趋势或 TR，用 5m 做顺势和小级别反转交易；也提到交易时可关注 15m/1h EMA。

执行框架：

- 1h / 4h：判断市场处于趋势、宽通道还是 TR。
- 15m / 1h EMA：作为日内方向和回调参考。
- 5m：寻找具体入场信号、突破、PB、失败突破、scalp 机会。

交易含义：

- 小周期信号如果逆大周期环境，要降低仓位或放弃。
- 大周期是 TR，小周期突破更容易失败。
- 大周期是强趋势，小周期逆势反转更容易失败。

工具提醒语：

> 1h/4h 是趋势还是 TR？5m 信号是否顺着大周期环境？是否正在用小周期信号硬扛大周期趋势？

一句话纪律：

> 大周期定交易环境，小周期只负责入场时机。

### 12.15 TR / Channel 中突破失败与止损空间

旧笔记中提到：TR 或通道中的突破大概率失败，适合考虑反向 scalp；如果做失败突破，止损需要留出足够空间，避免被假 BO 再次扫掉。

交易含义：

- TR / Channel 里的 BO 首先按高失败率处理，除非出现强 FT。
- 做失败突破时，不要把止损放得太近，因为价格常常会二次测试极端位。
- 止损放大不等于风险放大；需要同步降低仓位。

工具提醒语：

> 当前是否在 TR/Channel 中做失败突破？止损是否太近，容易被二次假突破扫掉？如果止损放大，仓位是否已经降低？

一句话纪律：

> TR/Channel 中突破先疑后信；做失败突破要给结构空间，并用仓位控制风险。

```yaml
    risk: 把普通回调误判为反转
    discipline_sentence: 2 bar reversal 必须结合位置。

  - id: multi_timeframe_environment_entry
    name: 大周期定环境，小周期找入场
    category: 多周期分析
    market_context: 1h, 4h, 15m, 5m
    trigger: 交易前分析
    checklist:
      - 1h/4h 是趋势还是 TR？
      - 5m 信号是否顺着大周期？
      - 是否正在用小周期信号硬扛大周期？
    action: 用 1h/4h 定环境，用 5m 找入场，逆大周期信号降级处理
    avoid: 只看 5m 单点信号
    risk: 小周期信号被大周期结构碾压
    discipline_sentence: 大周期定环境，小周期找入场。

  - id: tr_channel_failed_breakout_stop_room
    name: TR/Channel 中失败突破要给止损空间
    category: TR / 失败突破 / 风控
    market_context: Trading Range, Channel, failed BO, scalp
    trigger: 在 TR/Channel 中做失败突破或反向 scalp
    checklist:
      - 当前是否仍在 TR/Channel？
      - 突破是否无 FT？
      - 止损是否太近，容易被二次测试扫掉？
      - 如果止损放大，仓位是否降低？
    action: 用结构止损给二次测试空间，同时降低仓位控制风险
    avoid: 止损太近或放大止损但不降仓位
    risk: 被假突破二次扫损，或单笔风险失控
    discipline_sentence: TR/Channel 中突破先疑后信；失败突破要给结构空间。
```

## 14. 盈利管理：Take Profits and Let Runners Run

### 14.1 核心原则

> The name of the game is to take your profits and let runners RUN.  
> Stop letting green trades go red!!!

交易的目标不是每次都卖在最高点 / 买在最低点，而是在合理位置先兑现一部分利润，同时保留一部分仓位去吃可能出现的大行情。

这条规则要解决两个常见错误：

1. **完全不止盈**：浮盈时幻想更大行情，最后盈利单变亏损单。
2. **过早全平仓**：刚有一点利润就全部退出，导致真正的大行情没有仓位参与。

更好的做法是：

- 到达第一个合理目标位时，先兑现一部分利润；
- 剩余仓位作为 runner，让它跟随趋势继续运行；
- runner 必须有保护性止损，不能让已经明显盈利的交易重新变成亏损；
- runner 的止损可以根据结构、EMA、前一根强 K、趋势线、前高/前低或 breakeven 来移动。

### 14.2 什么时候应该先拿利润？

可以考虑在这些位置先兑现一部分利润：

- 到达 1R / 2R；
- 到达 Measured Move 目标位；
- 到达前高 / 前低；
- 到达区间上沿 / 下沿；
- 到达重要整数位；
- 到达 EMA / 趋势线附近；
- 出现 climactic move 或连续大 K 后开始停滞；
- 突破后没有继续 FT，开始重叠或反向吞回。

### 14.3 什么时候应该让 runner 跑？

可以保留 runner 的情况：

- 突破有好的 follow-through；
- 趋势中不断出现顺势强 K；
- 回调很浅，始终没有破坏趋势结构；
- 突破后没有回测突破点，甚至留下缺口；
- 逆势方被持续挤压，没有舒服退出机会；
- 当前行情可能走向 Measured Move 或第二段行情。

### 14.4 Stop Letting Green Trades Go Red

盈利单变亏损单通常不是技术问题，而是纪律问题。

交易中一旦出现明显浮盈，要问：

- 我是否已经到达第一个合理目标位？
- 我是否应该先平掉一部分？
- 剩余 runner 是否已经有保护性止损？
- 如果价格现在反转，我是否会后悔没有锁定利润？
- 我是不是因为贪心，才让盈利单没有任何保护？

执行原则：

- 一旦达到计划目标，至少要做一次利润管理决策；
- 如果已经部分止盈，runner 可以给行情空间；
- 如果没有部分止盈，就必须更严肃地保护浮盈；
- 不允许一笔已经明显盈利的交易，在没有重新评估的情况下变成亏损。

工具提醒语：

> 这笔交易已经是 green trade。你是否已经部分止盈或移动止损？不要让盈利单变红。

一句话纪律：

> 先拿一部分利润，留下 runner 追大行情；但 runner 必须受保护，不能让 green trade go red。

### 14.5 工具化规则

```yaml
- id: take_profit_and_let_runner_run
  name: 先兑现利润，同时保留 runner
  category: 出场管理 / 盈利保护
  market_context: 所有已有浮盈的交易，尤其是突破后有 FT、趋势行情、可能出现 MM 或第二段行情
  trigger: 持仓已经出现明显浮盈，或价格到达第一个合理目标位
  checklist:
    - 当前是否已经到达 1R/2R、MM、前高/前低、区间边缘、整数位或 EMA？
    - 是否已经部分止盈？
    - 剩余 runner 是否有保护性止损？
    - 行情是否仍有强 FT，值得保留 runner？
    - 如果现在反转，我是否会让盈利单变亏损？
  action: 到达合理目标位时先兑现部分利润；剩余仓位作为 runner，并用结构止损、breakeven 或 trailing stop 保护
  avoid: 完全不止盈、幻想卖在最高点、让明显浮盈无保护地变成亏损、过早全平导致错过趋势行情
  risk: 盈利单变亏损，或因过早全平错过真正的大行情
  discipline_sentence: Take profits and let runners run; never let a green trade go red without重新评估。
```

## 15. 交易纪律工具 / 网页应用设计 v0.1

> 目标：把本 Markdown 中的价格行为规则转成一个实际可用的交易纪律工具。核心不是“预测行情”，而是在交易前、交易中、交易后强制自己完成必要检查，减少遗漏、冲动交易、无止损下单、盈利单变亏损、止损后反手等纪律问题。

### 15.1 产品定位

这个工具应该被设计成一个 **Trading Discipline Copilot**，而不是自动交易机器人。

核心原则：

- **先规范行为，再考虑自动化。**
- **先手动记录和点击确认，再逐步接入交易数据。**
- **先提醒风险，不直接替你做交易决定。**
- **默认不自动下单，避免工具把纪律问题变成更大的自动化风险。**

它应该解决三个问题：

1. 下单前：是否真的有交易理由？是否符合价格行为规则？是否已经设置止损和目标？
2. 持仓中：是否已经到目标位？是否需要部分止盈、移动止损、保护 runner？是否正在让 green trade go red？
3. 平仓后：这笔交易是策略内结果，还是纪律性错误？下次需要避免什么？

---

### 15.2 核心工作流

应用主流程分成三个阶段：

```text
交易前检查 → 开仓后管理 → 平仓后复盘
```

每一笔交易都应该从一个 `Trade Session` 开始，而不是事后随便写笔记。

#### 阶段 A：交易前检查 Pre-Trade Checklist

目的：防止没有结构依据、没有止损、FOMO、报复交易、追无 FT 突破。

用户需要输入或点击：

- 交易品种：ES / NQ / 股票 / 加密货币等
- 方向：Long / Short
- 计划入场价
- 止损价
- 第一目标位
- 是否保留 runner
- 当前市场环境
  - 强趋势
  - 弱趋势 / 宽通道
  - Trading Range
  - Breakout Mode
  - Opening Range
  - Gap Open
  - 不确定
- 交易 setup
  - Breakout
  - Pullback
  - Failed Breakout
  - Reversal
  - H1/H2 或 L1/L2
  - Wedge / Double Top / Double Bottom
  - Inside Bar / Triangle
  - 其他

系统必须检查：

- 是否填写止损；
- 止损是否在合理结构位置；
- 是否有第一目标位；
- 盈亏比是否过低；
- 是否处在 TR 上沿追多 / TR 下沿追空；
- 突破是否有好的 FT；
- 是否刚刚被止损后想立刻反手；
- 当前 setup 是否和市场环境匹配。

工具提醒语示例：

> 你选择了 Breakout，但还没有确认 follow-through。请确认：这是有 FT 的突破，还是可能是假突破？

> 当前环境是 Trading Range，你正在区间上沿追多。请检查是否存在第二段陷阱或大阳线诱多。

> 你还没有设置止损。没有止损，不允许进入交易。

#### 阶段 B：开仓后管理 In-Trade Management

目的：防止有浮盈后无保护、防止盈利单变亏损、防止不按计划移动止损。

开仓后需要记录：

- 实际入场价；
- 实际仓位；
- 当前止损价；
- 当前目标位；
- 是否已经部分止盈；
- runner 是否仍在；
- 是否移动过止损；
- 当前最大浮盈 MFE；
- 当前最大浮亏 MAE。

系统提醒逻辑：

- 如果价格到达 1R / 2R / 前高前低 / MM / 区间边缘：提醒是否部分止盈；
- 如果已有明显浮盈但止损仍在原始位置：提醒保护利润；
- 如果部分止盈后 runner 没有保护性止损：提醒设置 runner stop；
- 如果价格接近 breakeven 且曾经大幅盈利：提醒不要让 green trade go red；
- 如果趋势仍强、有 FT、缺口未回补：提醒可以保留 runner，而不是过早全平。

工具提醒语示例：

> 这笔交易已经达到 1R。你是否要先兑现一部分利润，并让 runner 继续跑？

> 这笔交易曾经明显盈利，但现在接近回到入场价。不要让 green trade go red；请决定部分止盈、移动止损或退出。

> 突破后没有回测突破点并留下缺口，趋势可能很强。如果你已经部分止盈，可以考虑让 runner 继续运行，但必须有保护性止损。

#### 阶段 C：平仓后复盘 Post-Trade Review

目的：把交易结果拆成“市场结果”和“执行质量”，避免只用盈亏评价自己。

平仓后需要输入：

- 平仓价；
- 盈亏 R 倍数；
- 是否按计划入场；
- 是否按计划止损；
- 是否按计划止盈；
- 是否让 green trade go red；
- 是否止损后立刻反手；
- 是否违反了某条规则；
- 这笔交易属于：
  - 好交易，盈利；
  - 好交易，亏损；
  - 坏交易，盈利；
  - 坏交易，亏损。

复盘输出：

- 本笔纪律评分；
- 违反规则列表；
- 最主要错误；
- 下次提醒标签。

工具提醒语示例：

> 这笔交易亏损，但你遵守了计划和止损，属于策略内亏损。

> 这笔交易盈利，但你没有设置止损，属于坏交易盈利，不能奖励这种行为。

> 这笔交易最大浮盈达到 2R，最后变成亏损。请标记为 green-to-red 错误，并复盘盈利管理。

### 15.4 规则引擎设计

每条交易规则都可以转成机器可读结构。

```yaml
rule:
  id: breakout_needs_follow_through
  name: 突破要有好的跟随
  stage: pre_trade
  severity: warning
  trigger:
    setup: Breakout
  checklist:
    - 突破 K 是否足够强？
    - 突破后是否有 FT？
    - 是否快速回到区间内？
  block_if:
    - stop_loss_missing
  warn_if:
    - follow_through_not_confirmed
    - trading_range_extreme_chasing
  message: 突破不是信号，突破后的 FT 才是确认。
```

建议把规则分成三类：

1. **Blocker 规则**：违反就不允许继续。
   - 没有止损；
   - 没有入场理由；
   - 刚止损后立刻反手且没有新结构；
   - 仓位风险超过上限。

2. **Warning 规则**：允许继续，但必须确认风险。
   - TR 上沿追多 / TR 下沿追空；
   - 突破没有 FT；
   - 强 K 出现在区间极端位置；
   - 两次突破昨日高低点失败。

3. **Reminder 规则**：交易中提醒。
   - 到达 1R；
   - 到达 MM；
   - green trade 接近变 red；
   - runner 没有保护止损；
   - 突破后不回测并留下缺口。

### 15.6 自动化追踪设计

自动化建议分三阶段，不要一开始就做自动下单。

#### Phase 1：纯手动 MVP

- 用户手动输入交易计划；
- 点击 checklist；
- 手动标记入场、部分止盈、移动止损、平仓；
- 生成纪律评分和复盘。

优点：最安全、最快落地、能先验证流程是否真的有用。

#### Phase 2：半自动导入

- 从券商 / 交易平台导出 CSV；
- 导入成交记录；
- 自动匹配交易时间、方向、价格、盈亏；
- 自动生成 MFE / MAE 需要行情数据支持，可先不做。

适合用于复盘，不涉及下单风险。

#### Phase 3：实时监听但不自动下单

- 通过 broker API / webhook / 本地日志监听订单和持仓；
- 检测新订单是否有止损；
- 检测持仓是否达到 1R；
- 检测 green trade 是否接近变 red；
- 给出弹窗提醒或声音提醒。

原则：

- 工具可以提醒；
- 工具可以记录；
- 工具默认不自动交易；
- 自动改单 / 自动平仓必须作为高级功能，且需要单独确认和风控限制。

### 15.7 MVP 优先级

第一版不要做太大，建议只做 5 个核心功能：

1. **New Trade Checklist**：下单前检查，必须填写入场、止损、目标、setup、环境。
2. **Rule-based Alerts**：根据选择的 setup 和环境显示对应提醒。
3. **Open Trade Management**：记录是否部分止盈、移动止损、runner。
4. **Post-Trade Review**：平仓后复盘，标记错误和纪律评分。
5. **Daily Discipline Summary**：统计今日交易次数、违规次数、green-to-red 次数、止损后反手次数。

第一版先不要做：

- 自动下单；
- 复杂行情识别；
- 自动画线；
- 自动判断 TR / 趋势；
- 过度复杂的 AI 分析。

原因：现在最重要的是建立纪律闭环，而不是做一个复杂但用不起来的系统。

### 15.9 交易纪律评分

每笔交易不只看盈亏，还要算执行分。

建议初始满分 100：

- 无止损下单：-40
- 止损后立刻情绪反手：-30
- 明显 green trade 变 red 且没有重新评估：-25
- 突破无 FT 仍追单：-20
- TR 极端位置追强 K：-20
- 没有填写交易计划：-20
- 没有复盘：-10
- 按计划止损：+10
- 到目标位后部分止盈或保护 runner：+10
- 交易前完成 checklist：+10

评分解释：

- 90-100：高纪律交易；
- 70-89：基本合格，有小问题；
- 50-69：纪律不足，需要复盘；
- 50 以下：高风险情绪交易。

## 16. 工具化开发文档输出：Product Design + Codex Plan

基于本导图中的价格行为规则、交易前检查、持仓管理、止损止盈纪律、runner 管理和交易后复盘逻辑，已经拆分出两份开发文档：

1. `product_design.md`：定义 Trading Discipline Copilot 的产品定位、页面设计、核心工作流、规则引擎、数据模型、API 设计、自动化路线图和 MVP 验收标准。
2. `codex_implementation_plan.md`：定义给 Codex 使用的分阶段开发计划，从 Stage 0 到 Stage 11，一步一步实现网页应用，同时保留每阶段学习目标、测试命令和 commit 建议。

后续开发原则：

- 先实现手动 MVP，再考虑 CSV 导入或实时监听。
- 工具优先做纪律提醒，不做行情预测。
- 默认不自动交易，不自动平仓。
- 每个功能都要服务交易纪律闭环：交易前检查 → 持仓管理 → 平仓复盘 → 每日总结。
- Codex 每次只推进一个 Stage，避免一次性生成过大代码。

一句话原则：

> 先把交易纪律流程做成可点击、可记录、可复盘的工具，再逐步考虑自动化追踪。
